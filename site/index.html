Four stages, one readable path

Design principle

Make the first milestone useful, not theatrical.

The runtime is deliberately compact: parse telemetry, detect drift, choose a response, then write a forensic trail. It is not pretending to already have the zero-knowledge, formal methods, or swarm logic the research agenda aims for.

01

Ingest

Typed telemetry parsing keeps replay scenarios deterministic and easy to compare.

02

Detect

A rolling baseline converts cross-signal drift into an explainable anomaly score.

03

Respond

Policy logic scales mitigation based on severity while accounting for battery limits.

04

Audit

A chained run log preserves what happened, why it happened, and how the runtime reacted.
